# 知识库总览

欢迎使用 Yitwah 的个人知识库。这里收录了日常工作和生活中的学习笔记、实践总结和代码片段。

> **第一次使用？** 从右侧导航栏的"入门"开始，先看完快速上手再深入各板块。

## 文档使用指南

`★ Insight ─────────────────────────────────────`**
这个知识库采用 **docs.json 导航 + MDX 内容** 的双层架构：docs.json 控制"站点长什么样"（页面分组、tab 顺序），MDX 文件控制"内容是什么"。理解这个分离是掌握 Mintlify 的关键。
`─────────────────────────────────────────────────`

### 导航栏结构

整个站点分为四大板块，通过顶部 Tab 切换：

| Tab 名称 | 内容定位 |
|----------|---------|
| **文档** | 知识管理理念、工具使用、Markdown 规范 |
| **社交热点** | 每日社交媒体热点追踪（2026年4月） |
| **GitHub Trending** | GitHub 热门项目每日追踪（2026年3-4月） |
| **我的笔记** | 个人学习/工作/想法/代码等分类笔记 |

### 页面内的快速导航

- **左侧边栏**：当前 Tab 下的完整页面分组，可折叠/展开
- **右侧目录**：长页面自动生成的标题目录索引
- **搜索框**：右上角，输入关键词快速定位页面
- **Ask AI 按钮**：右下角 AI 助手，基于当前页面内容回答问题

### 如何贡献内容

**添加新笔记**（最常用）：

1. 切换到 **我的笔记** Tab
2. 在对应分类（学习/工作/想法/代码/研究/会议）下查看现有笔记
3. 通过 `/takenote` 命令可以快速追加新笔记（Claude Code 中可用）

**添加全新页面**（需要部署更新）：

1. 在对应目录下创建新的 `.mdx` 文件，添加 frontmatter
2. 在 `docs.json` 中注册路径（否则页面不会出现在导航中）
3. 提交 PR 或直接 push，部署后生效
