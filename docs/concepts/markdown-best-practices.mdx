# Markdown 最佳实践

Markdown 是一种轻量级标记语言，目标是让文档易于编写和阅读。本指南帮助你写出高质量的 Markdown 文档。

## 基本原则

### 1. 结构清晰

使用正确的标题层级：

```md
# 一级标题 - 页面标题
## 二级标题 - 主要章节
### 三级标题 - 子章节
```

建议最多使用到四级标题，过深的层级会降低可读性。

### 2. 段落与换行

- 段落之间留空行
- 不要用空格或 Tab 缩进代替列表
- 使用 `<br>` 标签来处理特殊换行需求

### 3. 列表规范

**无序列表**

```md
- 项目一
- 项目二
  - 嵌套项目
  - 嵌套项目
- 项目三
```

**有序列表**

```md
1. 第一步
2. 第二步
3. 第三步
```

## 代码块

### 行内代码

使用反引号包裹：`const x = 1`

### 代码块

指定语言以获得语法高亮：

````md
```javascript
function hello() {
  console.log('Hello, World!');
}
```
````

## 链接与图片

### 链接

```md
[链接文字](https://example.com)
```

### 图片

```md
![替代文字](/images/example.png)
```

## 表格

```md
| 表头1 | 表头2 | 表头3 |
|-------|-------|-------|
| 单元格 | 单元格 | 单元格 |
| 单元格 | 单元格 | 单元格 |
```

## 最佳实践建议

| 建议 | 说明 |
|------|------|
| 保持简洁 | 不要过度使用格式化 |
| 使用语义化标记 | 标题用于结构，不是样式 |
| 预览检查 | 提交前预览渲染效果 |
| 统一风格 | 保持文档风格一致 |

## MDX 扩展

如果使用 MDX，还可以使用 React 组件：

```mdx
<Callout type="warning">
  这是一条警告信息
</Callout>
```
