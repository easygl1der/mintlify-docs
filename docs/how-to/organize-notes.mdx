# 整理笔记的方法

好的笔记组织方式能让知识库保持长期可用，避免成为"知识垃圾场"。

## 分类原则

### 按主题而非来源分类

**按主题**（推荐）：

```
笔记/
├── 编程/
│   ├── Python/
│   ├── JavaScript/
│   └── DevOps/
├── 设计/
│   └── UI-原则/
└── 管理/
    └── 项目管理/
```

**按来源**（不推荐）：

```
笔记/
├── 公众号文章/
├── B站视频/
└── 书籍笔记/
```

按主题分类更容易找到相关内容。

## 文件命名

### 推荐格式

```
YYYY-MM-DD-笔记标题.md
2024-01-15-Python虚拟环境完全指南.md
```

### 避免

- 中文标题（部分工具支持不好）
- 特殊字符
- 过长的文件名

## Frontmatter 规范

每个笔记应包含清晰 frontmatter：

```yaml
---
title: 笔记标题
description: 简短描述
tags: [标签1, 标签2]
created: 2024-01-15
updated: 2024-01-20
---
```

## 双向链接

使用双向链接连接相关笔记：

- Obsidian: `[[相关笔记]]`
- 其他工具: 使用标签系统

### 好处

- 发现相关内容
- 构建知识网络
- 避免信息孤岛

## 定期整理

建议建立整理习惯：

| 频率 | 内容 |
|------|------|
| 每天 | 快速归类新笔记 |
| 每周 | 回顾本周收集的内容 |
| 每月 | 清理过时内容，整理结构 |

## MOC (Map of Content)

对于大型主题，创建 MOC 页面作为入口：

```md
# Python MOC

这是 Python 相关内容的总览。

## 入门
- [[Python基础]]

## 进阶
- [[Python高级特性]]
- [[Python性能优化]]

## 生态
- [[Python框架推荐]]
```

## 总结

整理笔记的核心原则：

1. **一致性** - 保持统一的命名和组织方式
2. **相关性** - 相关内容放在一起
3. **简洁性** - 避免过度分类
4. **迭代** - 定期回顾和优化结构
