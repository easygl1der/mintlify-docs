## 2026-04-08 08:30

### Q：如何将 Mintlify AI Assistant 与 Comet 右键功能结合？

**A：结合思路**

Mintlify 本身有 AI Assistant（chat copilot），Comet 是浏览器扩展提供"选中文字 → 定义/查来源"等右键能力。两者结合可以：

1. **在 Mintlify 文档页右键 AI 查询** — 选中代码/术语 → Comet 触发 Mintlify AI 分析
2. **Mintlify AI 的能力延伸到浏览器全局** — 通过 Comet 调用 Mintlify 的 API
3. **获取示例代码** — 从 Mintlify GitHub 仓库 (mintlify/mintlify) 的 `examples/` 目录找参考实现

**示例代码获取路径**：
- Mintlify 官方 examples: https://github.com/mintlify/examples
- Mintlify 主仓库组件示例: https://github.com/mintlify/mintlify/tree/main/components

**Comet 功能参考**：
- 选中文字 → 右键"定义"（调用 AI 解释）
- 选中文字 → 右键"查来源"（web search）
- 可扩展：添加"Mintlify AI 分析"到右键菜单

**实施步骤**：
1. 从 Mintlify GitHub 获取 AI Assistant 的 API 接口文档
2. 在 Comet 扩展中添加自定义右键项（调用 Mintlify API）
3. 文档化部署流程

### Q：Mintlify 源码体系深度解析

**核心发现**：Mintlify 公开的是"半开源产品"，只有前台能力可见，后台商业系统未公开。

**公开仓库价值排名**：

| 仓库 | 学习价值 | 说明 |
|------|----------|------|
| `mintlify/components` | ⭐⭐⭐⭐⭐ | React UI 组件核心，Storybook 在线预览，pnpm monorepo，TS 95%+ |
| `mintlify/mdx` | ⭐⭐⭐⭐ | MDX 解析链路，内容渲染内核 |
| `mintlify/docs` | ⭐⭐⭐⭐ | 官方文档站源码，docs.json 配置，AI/LLM/Agent 功能文档 |
| `mintlify/astro` | ⭐⭐⭐ | Astro 集成，平台无关化战略，peer 依赖 components |
| `mintlify/assistant-embed-example` | ⭐⭐⭐ | AI 助手嵌入教程，Vercel AI SDK v4，接入方式最清晰 |
| `mintlify/starter` / `themes` | ⭐⭐ | 起步模板，主题配置 |
| `mintlify/docs-hermine` | ⭐ | 自动生成的客户文档（无技术含量） |

**架构层次**：

```
docs.json + *.mdx 内容
         ↓
  mintlify/astro（框架集成层）
         ↓
  mintlify/components（UI 渲染层，核心基座）
         ↓
   最终的文档站页面
```

**关键洞察**：
- **`components` 是整个生态的底层 UI 基座**，所有框架集成都依赖它
- **Mintlify 是"文档站 SaaS"，非全开源** — 后台（计费/权限/部署/CDN/AI 编排）不公开
- **`assistant-embed-example`** 透露产品方向：AI 助手可嵌入式，不绑定的 Mintlify 平台
- **`docs-hermine`** 无意暴露了 Mintlify 的自动化机制：为每个用户在 GitHub org 下创建文档仓库
- 工程规范：`.claude/`/`.cursor/rules` + Vale lint + Husky hooks，AI 编程工具深度集成

**下一步研究**：`mintlify/components` 源码 + `@mintlify/astro` 实现链路
