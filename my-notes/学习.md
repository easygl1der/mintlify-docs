## 2026-04-08 理解 SaaS 管理后台：前台与后台的区别

> 来源：`/Users/yueyh/Projects/WSL/MyNotes/4_学习/理解_SaaS_管理后台_前台与后台的区别.md`

### 核心概念

**SaaS = Software as a Service（软件即服务）**

常见例子：Slack、Notion、Salesforce、Figma、钉钉、企业微信。传统软件需要安装到本地，SaaS 通过浏览器或 App 直接访问，数据存在云端，按订阅收费。

**前台 vs 后台**

- 前台 = 用户干活用的界面（普通员工、客户使用的部分）
- 后台 = 管数据、配置、统计的界面（管理员使用的部分）

两者通过同一个数据库连通，但面向不同角色、展示不同功能。

### 具体例子

#### 微信公众号

| 角色 | 看到的界面 |
|------|-----------|
| 普通用户 | 关注公众号后，看文章、留言、买商品 |
| 运营者（后台） | 登录 mp.weixin.qq.com，发文章、看数据、设置自动回复、管理粉丝 |

#### 淘宝商家

| 角色 | 看到的界面 |
|------|-----------|
| 买家（前台） | 淘宝 App/网页，搜索商品、下单支付 |
| 卖家/商家（后台） | 商家后台，上架商品、处理订单、看生意参谋 |

#### shadcn/admin 项目（纯后台模板）

这是一个"纯后台"模板，前台可能是：

| 页面 | 前台（用户用） | 后台（管理员用） |
|------|-------------|---------------|
| 任务系统 | 员工看自己的任务列表 | 管理员创建/分配/统计所有任务 |
| 用户管理 | - | 管理员增删改查用户账号 |
| 销售报表 | - | 管理员看收入数据 |

如果要做完整 SaaS，前台大概会有：官网首页、定价页、客户控制台、登录/注册。

## LaTeX fancy 环境：sleek-template 源码分析

> 来源：[francois-rozet/sleek-template](https://github.com/francois-rozet/sleek-template)
> 分析方式：Agent Team 抓取 .tex + .sty 文件

### 核心包依赖

| 包 | 作用 |
|----|------|
| `amsthm` | 基础 theorem 基础设施（`\newtheorem`、`\theoremstyle`） |
| `thmtools` | 增强 theorem 命令（`\declaretheorem`） |
| `mdframed`（TikZ framemethod） | 有框 theorem 变体 |

### 关键机制：`mdtheorem` 自定义 key

```latex
\define@keyythm}{mdthm}[{}]{
    \thmt@trytwice{\def\thmt@theoremdefiner{\mdtheorem[#1]}}{}
}
```

通过 `\declaretheorem[mdthm={...}]` 将样式选项透传给 `mdtheorem`，创建带 `mdframed` 边框的 theorem。

### 环境分类与配色

| 环境 | 边框/背景颜色 | 计数器 |
|------|--------------|--------|
| `theorem`、`lemma` | 红色 `red!20` | 共享 `theorem` 计数器 |
| `proposition` | 蓝色 `blue!20` | 从属 `theorem` |
| `definition` | 绿 `black!20!green!20` | 独立计数器 |
| `hypothesis` | 蓝色 `blue!20` | 独立计数器 |
| `question` | 橙色 `orange!20` | 独立计数器 |
| `answer` | 灰色（默认） | 从属 `question` |
| `proof` | 灰色（默认） | 从属 `theorem` |
| `example`、`method` | 灰色（默认） | 独立计数器 |

### 基础样式定义（`thicc`）

```latex
\mdfdefinestyle{thicc}{
    linewidth=0.25em,
    roundcorner=0.15em,
    linecolor=black!10,
    frametitlebackgroundcolor=black!10,
    nobreak=true,
    skipabove=\topskip,
    skipbelow=\topskip,
    innerleftmargin=0.5em,
    innerrightmargin=0.5em,
}
```

### 快速上手模板

**无框版本（amsthm 直接定义）**：

```latex
\usepackage{amsthm}
\theoremstyle{plain}
\newtheorem{theorem}{Theorem}
\newtheorem*{remark}{Remark}   % 无编号

\theoremstyle{definition}
\newtheorem{question}{Question}

\theoremstyle remark
\newtheorem{answer}[question]{Answer}   % 附属 question
```

**有框版本（mdframed + thmtools）**：

```latex
\usepackage[framemethod=TikZ]{mdframed}
\usepackage{amsmath,thmtools}

% 定义 key
\define@keyythm}{mdthm}[{}]{
    \thmt@trytwice{\def\thmt@theoremdefiner{\mdtheorem[#1]}}{}
}

% 声明有框 theorem
\declaretheorem[
    mdthm={style=thicc, linecolor=red!20, frametitlebackgroundcolor=red!20},
    name=Theorem,
    sibling=theorem
]{fmd-theorem}

% 橙色 question
\declaretheorem[
    mdthm={style=thicc, linecolor=orange!20, frametitlebackgroundcolor=orange!20},
    name=Question
]{fmd-question}

% answer 附属 question，可分页
\declaretheorem[
    mdthm={style=thicc, nobreak=false},
    name=Answer,
    within=fmd-question
]{fmd-answer}
```

### 关键命令速查

| 命令 | 包 | 说明 |
|------|----|------|
| `\newtheorem{name}{label}` | amsthm | 创建编号环境 |
| `\newtheorem*{name}{label}` | amsthm | 创建无编号环境 |
| `\theoremstyle{plain\|definition\|remark}` | amsthm | 设置后续 theorem 样式 |
| `\declaretheorem[mdthm={...}]{name}` | thmtools+mdframed | 创建带框 theorem |
| `\mdfdefinestyle{name}{...}` | mdframed | 定义 mdframed 样式 |

### `question`/`answer` 成对使用

```latex
% question 独立计数
\newtheorem{question}{\lgquestion}

% answer 附属 question，重新定义 proof 环境
\let\proof\relax\let\endproof\relax
\newtheorem{answer}{\lganswer}[question]

% 有框版本
\declaretheorem[mdthm={style=thicc, linecolor=orange!20}, name=Question]{fmd-question}
\declaretheorem[mdthm={nobreak=false}, name=Answer, within=fmd-question]{fmd-answer}
```

> 模板无 .cls 文件，直接基于标准 `report`/`article` 类，用 `.sty` 包扩展。配色通过 `mdframed` 的 `linecolor` 和 ` frametitlebackgroundcolor` 控制。`\theoremstyle` 三种（plain/definition/remark）分别控制定理类、定义类、证明/备注类的字体和间距。

## 2026-04-13 marimo 响应式 Python 编程环境

> 来源：https://docs.marimo.io/#a-reactive-programming-environment

### 核心定位

**marimo** 是一个响应式 Python 编程环境，旨在成为 Jupyter Notebook 的现代替代品。

### 核心特性

- **响应式执行模型**：代码单元修改后，所有依赖该单元的 downstream 单元**自动重新执行**，无需手动干预
- **可重现性**：所有 notebook 都是纯 Python 脚本，可通过 CLI 执行
- **交互式 UI 组件**：内置滑坡条、下拉菜单、复选框等，与 Python 状态自动绑定
- **Git 友好**：notebook 存储为 .py 文件，而非 .ipynb JSON 格式
- **部署简单**：可通过 `marimo run` 或打包为 HTML 分享

### 与 Jupyter Notebook 的核心区别

| 特性 | marimo | Jupyter |
|------|--------|---------|
| 执行模型 | 响应式（reactive） | 命令式（imperative） |
| 状态同步 | 自动追踪依赖 | 手动 kernel 重启 |
| 存储格式 | .py 纯文本 | .ipynb JSON |
| 版本控制 | Git 友好 | 合并冲突困难 |
| UI 交互 | 内置组件 | 需 ipywidgets |

### 使用场景

- **数据探索**：响应式特性使迭代分析更流畅
- **教学演示**：代码修改即时反馈，便于讲解
- **可交互报告**：幻灯片式展示，嵌入 UI 组件

### 安装与使用

```bash
pip install marimo
marimo run your_script.py
```

## 2026-04-13 开源 AI Web Browser / Browser Agent

> 来源：WebSearch 调研

### 主要项目一览

| 项目 | 类型 | 本地 AI | 平台 | 定位 |
|------|------|---------|------|------|
| **BrowserOS** | 完整浏览器（Chromium 分支） | ✅ Ollama/LM Studio | macOS/Win/Linux | 即装即用的 AI Chrome 替代品 |
| **Browser-Use** | 开发框架/库 | ✅ Ollama | 全平台 | 最流行的 AI 浏览器控制框架，50k+ GitHub stars |
| **Nanobrowser** | Chrome 扩展 | ✅ | Chrome | 多 Agent 工作流，免费 OpenAI Operator 替代 |

### 选型建议

- **想直接当浏览器用** → **BrowserOS**（Chromium 核，Chrome 扩展兼容）
- **想开发 AI Agent 控制浏览器** → **Browser-Use**（最成熟，生态最广）

### Browser-Use 技术细节

- 基于 Playwright / Puppeteer 控制浏览器
- 支持 OpenAI / Google / Ollama 等多种 LLM provider
- 核心思路：让 LLM 生成操作序列（click、type、scroll 等）执行任务

### 相关项目补充

- **Fellou** — AI 浏览器 + Eko 2.0 框架
- **Open WebUI** — 不是浏览器，是自托管 AI 界面，可完全离线运行

## 2026-04-21 SearXNG 隐私搜索引擎

### SearXNG vs 普通浏览器搜索

| 维度 | 普通浏览器（Google/Bing/DuckDuckGo） | SearXNG |
|------|-----------------------------------|----------|
| **隐私** | 追踪搜索记录，构建用户画像 | 不追踪，无广告追踪 |
| **数据** | 数据留在大厂 | 不存储任何数据 |
| **来源** | 单一搜索引擎 | 聚合 50+ 搜索引擎 |
| **结果** | 可能被过滤/信息茧房 | 多源聚合结果 |

### SearXNG 适用场景

**1. 隐私保护**
- 搜索不被追踪，无个性化广告
- 无需账号

**2. 结果多元化**
- 同时获取 Google、Bing、DuckDuckGo 结果
- 避免信息茧房

**3. 绕过审查**
- 聚合多源，一个被屏蔽不影响整体

**4. 专业搜索**
- 学术：arxiv、Google Scholar、Semantic Scholar
- 代码：GitHub、Stack Overflow
- 图片/视频/新闻：多引擎聚合

- 运行时间：31+ 小时
- 访问地址：http://127.0.0.1:8080
