## 二、四框架本质差异：图灵完备 vs 流程式

### 两种编排哲学

| 哲学 | 代表框架 | 本质 | 适用场景 |
|------|---------|------|---------|
| **图灵完备状态机** | LangGraph | 有向图 + 显式状态 + 条件边，可表达任意复杂流程 | 复杂分支、循环、容错恢复、断点续跑 |
| **流程式编排** | CrewAI、OpenAI Agents SDK | 预定义顺序/层级/Handoff，假设流程可预期 | 线性任务链、快速原型、团队协作流水线 |
| **对等对话式** | AutoGen | Agent 间共享消息池，Manager 仲裁轮流发言 | 模拟多人开会、多角色头脑风暴 |

### LangGraph 的护城河

LangGraph 是**唯一**在"图灵完备状态机 + checkpoint 断点恢复 + 原生 streaming + human-in-the-loop"四个维度同时原生支持的框架。

这四个能力叠加在一起，解决的是一个根本问题：**你的 Agent 系统能不能在生产环境里真正长期跑起来不出事故**？

- 没有 checkpoint：Agent 执行到一半崩了，全部重来
- 没有 human-in-the-loop：危险操作无法人工审核
- 没有 streaming：前端感知不到 Agent 在干什么
- 没有状态机：复杂业务逻辑全靠 Agent 自己推理，不稳定

## 三、针对你需求的选型建议

### 需求拆解

```
PDF 阅读      → 文档解析、文本提取、向量化、RAG
AI 问答      → 多轮对话、上下文记忆、引用溯源
IDE 式工作台 → 多面板布局、实时状态同步、断点续跑、human-in-the-loop
多 Agent     → PDF 解析 Agent + Q&A Agent + 笔记 Agent（未来扩展）
Harness      → 可观测（tracing）、多 session 管理、工具标准化
```

### 最终结论

**主框架：LangGraph**
**辅助：OpenAI Agents SDK（简单线性流）**
**原型参考：AutoGen Magentic-One**
**快速验证：CrewAI（仅限流程简单时）**

```
┌─────────────────────────────────────────────────────────────┐
│  为什么是 LangGraph 而不是别的？                              │
├─────────────────────────────────────────────────────────────┤
│ 1. Checkpoint 断点恢复 = IDE 工作台的命                       │
│    用户在 AI 生成笔记时刷新页面，能从断点恢复而不是重来         │
│                                                              │
│ 2. StateGraph 显式状态 = 调试友好                            │
│    你知道当前在哪个节点，不依赖 LLM 自我判断                   │
│                                                              │
│ 3. Streaming 按节点推送 = 多面板 UI 的天然数据源              │
│    每个节点完成 → 前端对应面板更新，不是等整个任务结束         │
│                                                              │
│ 4. human-in-the-loop = 危险操作可人工审核                    │
│    AI 要删文件/发消息，先暂停等用户确认                         │
│                                                              │
│ 5. 生态完整 = Langfuse+pgvector+FastAPI 一条龙               │
│    不用自己拼凑，每一层都有成熟集成                            │
└─────────────────────────────────────────────────────────────┘
```

### AutoGen 的正确打开方式

⚠️ **重要警报**：AutoGen 微软已推维护模式，新用户建议用 Microsoft Agent Framework。

但 Magentic-One（微软开源的多 Agent 系统）的架构值得精读——它解决的就是"PDF + 文件处理 + 代码执行 + 多 Agent 协作"，和你需求高度重叠。

### CrewAI 的正确打开方式

适合：团队需要**最快出成果**、流程相对线性（调研→分析→写作）、没有图论基础的团队。

不适合：需要精细状态机控制的 IDE 场景；CrewAI 的黑盒状态管理在复杂场景下不够灵活。

| 框架 | 一句话定性 | 适合你吗 |
