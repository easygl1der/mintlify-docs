### 关键配置

**settings.yml 核心设置**：
- `secret_key`: 硬编码值（生产环境应使用 `SEARXNG_SECRET` 环境变量）
- `base_url`: `false`（默认）
- `image_proxy`: `false`
- `method`: `POST`
- `server.port`: `8888`（容器内），`8080`（暴露端口）

**启用的搜索引型**（55+）：
Google、Bing、Wikipedia、GitHub、Stack Overflow、arxiv 等

### 验证流程

1. **容器状态检查**
```bash
docker compose ps
```

2. **HTTP 健康检查**
```bash
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8080/
# 返回 200 即正常
```

3. **搜索功能测试**
```bash
curl -s "http://127.0.0.1:8080/search?q=test"
# 检查是否返回 HTML 结果页
```

4. **Agent Team 验证**（OMC 工作流）
- 使用 `executor` agent 测试搜索 API
- 使用 `executor` agent 读取 settings.yml 验证配置
- 关闭 limiter 以支持 JSON API 测试

### 访问地址

- **本地访问**：http://127.0.0.1:8080
- **浏览器打开**：需临时移除 `~/.agent-browser/config.json` 中的 proxy 配置

### 注意事项

1. **Limiter 导致 JSON API 403**
   - 原因：`server.limiter: false` 但仍有 CSRF 保护
   - 解决：使用 HTML 格式或添加 proper headers

2. **agent-browser proxy 问题**
   - `~/.agent-browser/config.json` 中的 `proxy` 配置会干扰本地访问
   - 解决：临时移除或使用 `AGENT_BROWSER_PROXY=""` 覆盖

3. **Git 忽略 .env 文件**
   - `.env` 包含敏感配置，已在 `.gitignore` 中

## Mac Mini 外置 SSD 路径映射（Symlink）配置

**日期**：2026-04-06

---

### Q：如何让 ~/Projects 的访问落到外置 SSD 上？

**通过 symlink（符号链接）实现路径映射**：
```bash
mv ~/Projects ~/Projects_bak                              # 备份原目录
ln -s /Volumes/SSK\ SSD/Projects ~/Projects              # 创建 symlink
```
- **`~/Projects` → `/Volumes/SSK SSD/Projects`**
- 应用无感知，完全透明，读写删都落在 SSD 上

---

### Q：rm 删除 ~/Projects 下的文件会怎样？是否能在 Volumes 删除？

**两种路径删的是同一个真实文件**：

| 操作 | 删的是 |
|------|--------|
| `rm ~/Projects/foo.txt` | **SSD 上真实文件** |
| `rm /Volumes/SSK SSD/Projects/foo.txt` | **SSD 上真实文件** |

**没有区别**，symlink 对文件操作完全穿透。

> 但 `rm ~/Projects`（不带文件名）删的是 symlink 本身，真实文件不受影响。

---

### Q：什么操作不会穿透 symlink？

| 命令 | 行为 |
|------|------|
| `rm ~/Projects` | **删 symlink 本身**，真实文件不受影响 |
| `du -sh ~/Projects` | 只计算 symlink 大小（几字节） |
| `du -shL ~/Projects` | 计算真实文件大小 |
| `mv ~/Projects ~/foo` | 移动 symlink，真实目录不受影响 |

---

### Q：如何查看当前路径是逻辑路径还是真实路径？

```bash
pwd        # 逻辑路径（/Users/yueyh/Projects）
pwd -P    # 真实物理路径（/Volumes/SSK SSD/Projects）
```

---

### Q：什么操作是危险的高风险操作？

**直接对 SSD 真实目录操作**：
```bash
rm -rf /Volumes/SSK\ SSD/Projects/*   # ← 直接删 SSD 真实文件
```
**记住原则**：`~` 开头操作 symlink（相对安全）；`/Volumes/SSK SSD/` 开头操作真实文件（危险）。

### 修复 JSON API 403 错误 (2026-04-21 13:10)

**问题**：Onyx 连接 SearXNG 时返回 403，JSON 格式未启用

**原因**：`search.formats` 只配置了 `html`，缺少 `json`

**修复步骤**：
1. 编辑 `core-config/settings.yml`
```yaml
search:
  formats:
    - html
    - json  # 添加这行
```
2. 重启容器
```bash
docker restart searxng-core
```

**验证**：
```bash
curl "http://127.0.0.1:8080/search?q=test&format=json"
# 返回 JSON 结果 ✅
```
