## Mac Mini 外置 SSD 路径映射（Symlink）配置

**时间**：2026-04-06

### 背景
Mac Mini 内置盘（Data 卷 228GB）使用率 98%，外置 SSD "SSK SSD"（500GB）基本闲置。通过 symlink 将用户目录映射到 SSD。

### 磁盘现状
| 磁盘 | 容量 | 已用 | 可用 |
|------|------|------|------|
| 内置 Data 卷 | 228 GB | 187 GB → 160 GB | 5.1 GB → 32 GB |
| SSK SSD | 500 GB | 14 MB → 20 GB | 466 GB |

### 关键操作记录
1. 备份：`rsync -avh --progress ~/Projects/ /Volumes/SSK SSD/Projects_backup/`
2. 改名：`mv ~/Projects ~/Projects_bak`
3. 创建 symlink：`ln -s /Volumes/SSK SSD/Projects ~/Projects`
4. **当前状态**：`~/Projects` → `/Volumes/SSK SSD/Projects`
5. **备份位置**：`/Users/yueyh/Projects_bak`（原始目录，可确认后删除）

### Symlink 穿透性规则

#### ✅ 穿透（完全透明）
- 读写删：文件操作落在 SSD 上
- `cd`、`ls`、`cat`、`open`：正常穿透
- 编译器/编辑器：正常工作
- `rsync`：默认穿透，复制真实文件

#### ⚠️ 不穿透（特殊情况）
| 命令 | 行为 |
|------|------|
| `rm ~/Projects` | 删除 symlink 本身，真实文件不受影响 |
| `rm -rf ~/Projects/*` | 删除 SSD 上真实文件 |
| `du -sh ~/Projects` | 只计算 symlink 大小（几字节） |
| `du -shL ~/Projects` | 计算真实文件大小 |
| `mv ~/Projects ~/foo` | 移动 symlink，真实目录不受影响 |
| Time Machine | 可能只备份 symlink 而不跟踪文件 |

#### ⚠️ 危险操作（直接作用真实文件）
- `rm -rf /Volumes/SSK SSD/Projects/*` → 直接删 SSD 上真实文件
- 任何以 `/Volumes/SSK SSD/` 开头的操作直接作用在真实文件上

### 查看真实路径命令
```bash
pwd        # 逻辑路径（symlink 显示 /Users/yueyh/Projects）
pwd -P    # 真实物理路径（穿透后是 /Volumes/SSK SSD/Projects）
readlink ~/Projects  # 显示 symlink 指向
```

### 清理缓存命令（未来可用）
```bash
rm -rf ~/.cache/*
rm -rf ~/Library/Caches/*
npm cache clean --force
```

### 适用场景
Mac Mini 内置盘满，外置 SSD 有余量时使用。可迁移的目标：
- `~/Projects`（代码，最稳定推荐）
- `~/.cache`（缓存，清理比迁移更彻底）
- `~/.npm`（npm 缓存，可清理）
- `~/Library/Application Support/` 下的部分文件夹（需谨慎）
