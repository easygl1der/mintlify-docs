## 项目简介

oh-my-codex 是一个基于人工智能的代码生成工具，旨在为开发者提供智能化的代码辅助功能。该项目同时支持 OpenAI Codex 和 Ollama 两个 AI 后端，用户可根据本地部署或云端服务的需求灵活选择。这种双后端设计体现了良好的架构灵活性，使项目既适用于追求高精度的云端场景，也支持注重隐私保护的本地部署场景。

作为一个小型的 Python 工具项目，oh-my-codex 采用轻量级设计理念，专注于解决代码生成这一核心需求，避免了过度工程化的问题。项目遵循 Python 社区的标准实践，包括使用 setup.py 进行包分发、通过 dotenv 管理环境变量等。

## 技术亮点

### 1. 双后端支持架构

项目同时支持 OpenAI Codex 和 Ollama 两个 AI 后端，这一设计具有重要的实用价值：

- **OpenAI Codex**：适合追求高精度的生产环境，提供业界领先的代码生成能力
- **Ollama**：适合注重隐私或需要本地部署的场景，支持开源大语言模型

用户可根据实际需求灵活切换，无需维护两套独立工具。

### 2. 环境变量隔离机制

项目使用 `python-dotenv` 管理敏感配置（如 API Key），遵循以下安全最佳实践：

- 敏感信息存储在 `.env` 文件中，与代码分离
- `.env.example` 提供配置模板，方便新用户快速上手
- 避免硬编码敏感信息，降低代码泄露风险

### 3. 轻量级设计理念

项目始终保持极简设计原则：

- 仅使用三个核心依赖，无冗余库
- 代码规模控制在 500-800 行，结构清晰
- 功能聚焦，专注解决代码生成这一核心需求
- 无过度工程化，降低学习和维护成本

### 4. 标准化打包规范

采用 `setup.py` 进行包分发，符合 Python 社区规范：

- 支持 `pip install -e .` 开发模式
- 便于发布到 PyPI
- 入口点定义清晰
