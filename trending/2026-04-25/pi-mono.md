## 项目简介

**pi-mono** 是由 libGDX 框架创始人 **badlogic** 维护的极简化 Maven POM（Project Object Model）仓库项目，专门用于解决 libGDX 游戏引擎在 Raspberry Pi（树莓派）嵌入式平台上的后端依赖配置问题。

### 核心理念

该项目贯彻**极简主义（Minimalist）**设计哲学，仅通过 2 个文件（`README.md` + `pom.xml`）解决 libGDX 在树莓派平台上的跨架构部署难题。这种"一个问题，一个解决方案"的设计思路，体现了作者对职责单一原则（Single Responsibility Principle）的深刻理解。

### 目标平台

| 架构标识 | 适用设备 | 特性支持 |
|----------|----------|----------|
| `armv6-hardfloat` | Raspberry Pi 1 / Zero / Zero W | ARMv6 指令集 + 硬浮点 ABI（armhf） |
| `armv7-hardfloat` | Raspberry Pi 2 Model B | ARMv7 指令集 + 硬浮点 ABI |
| `armv7-hardfloat+neon` | Raspberry Pi 2/3/4 (32-bit) | ARMv7 + NEON SIMD 矢量指令优化 |

### 技术创新点

#### 1. 单 POM 多模块模拟

通过单个 `pom.xml` 文件整合多个架构配置，减少配置文件数量的同时保持结构清晰。这种设计在 Maven 项目中属于较新颖的实践。

#### 2. 硬浮点 ABI 规范遵循

项目严格遵循 ARM 硬浮点 ABI（armhf）标准，相比软浮点（softfp）具有以下优势：

```c
// 硬浮点 vs 软浮点性能对比（理论值）
硬浮点 (armhf): 浮点运算直接在 FPU 执行，吞吐量高
软浮点 (softfp): 浮点运算模拟，延迟高
性能提升: 约 2-5x（取决于计算密集度）
```

#### 3. NEON SIMD 向量化支持

为 Raspberry Pi 2+ 提供 NEON SIMD 指令集优化选项，可显著提升向量/矩阵运算性能：

```c
// NEON 可优化的典型运算
- 图形变换（矩阵乘法）
- 物理模拟（碰撞检测）
- 音频处理（FFT）
- 图像处理（滤镜、缩放）
```
