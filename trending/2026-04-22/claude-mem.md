## 项目简介

**claude-mem** 是一个功能性的 Emacs Lisp 扩展包，旨在将 Anthropic 的 Claude AI 用作持久化的外部记忆系统。该项目巧妙地将现代 AI 向量搜索技术与 Emacs 的强大可扩展性相结合，为用户提供语义化的记忆存储和检索能力。

### 核心价值

1. **语义记忆管理** - 利用 Claude 嵌入模型实现自然语言理解和相似度匹配
2. **本地向量存储** - 使用 SQLite 配合向量索引，数据完全保存在本地
3. **隐私优先设计** - 记忆数据不上传云端，仅使用 API 进行语义分析
4. **Emacs 原生体验** - 无缝集成 M-x 命令、consult 等 Emacs 生态工具

### 目标用户

- Emacs 重度用户和爱好者
- 需要 AI 辅助记忆管理的开发者和知识工作者
- 追求个人知识管理数字化的专业人士

## 潜在问题

### 高优先级风险

| 风险 | 描述 | 影响 | 建议 |
|------|------|------|------|
| **API 密钥泄露** | 用户可能在配置文件中明文存储 API 密钥 | 高 - 安全风险 | 建议使用 `auth-source` 或环境变量 |
| **API 调用成本** | 嵌入向量生成需要 API 调用 | 中 - 财务风险 | 添加批量处理和缓存机制 |
| **网络依赖** | 完全依赖外部 API 服务 | 中 - 可用性风险 | 考虑离线模式支持 |

### 中优先级问题

| 问题 | 描述 | 影响 |
|------|------|------|
| **单点故障** | claude-engine 出现问题将影响整个系统 | 功能可用性 |
| **API 版本锁定** | 使用特定嵌入模型，可能存在兼容性问题 | 长期维护 |
| **Emacs 29.1+ 要求** | 排除部分使用旧版本的用户 | 用户覆盖范围 |

### 安全性分析

```elisp
; ⚠️ 当前配置方式 (存在风险)
(setq claude-mem-api-key "sk-ant-...")  ; 明文存储在配置文件中

; ✅ 推荐配置方式 (更安全)
(setq claude-mem-api-key 
      (or (getenv "ANTHROPIC_API_KEY")
          (auth-source-pick-first-password :host "api.anthropic.com")))
```
