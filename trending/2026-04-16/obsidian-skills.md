## 项目简介

obsidian-skills 是由 @kepano 维护的 Obsidian.md 应用官方 CSS 源代码框架项目。该项目的主要目的是向开源社区展示 Obsidian 界面的样式实现方式，并文档化设计决策。作为 Obsidian 应用的核心样式代码库，它负责构建在桌面端、移动端以及 Publish 网站上呈现的样式界面。

该项目采用模块化设计理念，将 CSS 代码按功能领域拆分为多个独立的样式模块，每个模块负责特定的功能区域，如应用程序框架、工作区布局、编辑器样式、主题变量等。这种设计使得代码结构清晰、职责分明，便于维护者理解和修改。

需要特别说明的是，obsidian-skills 项目的官方声明明确指出：Obsidian.css 不是 Obsidian 的官方产品。尽管它提供了 Obsidian 界面样式的参考实现，但在实际使用中需要开发者自行适配和验证兼容性。

### 项目结构特点分析

#### 1. 模块化架构设计

项目采用严格的模块化架构设计，每个 CSS 文件都专注于特定的功能领域。这种设计的优势在于：

- **职责单一**：每个模块只负责一个功能领域，便于理解和维护
- **独立性强**：模块之间耦合度低，可以独立修改单个模块而不影响其他部分
- **可复用性**：核心样式模块可以在不同场景下复用

#### 2. 跨平台样式覆盖

项目实现了三种平台的样式支持，确保一致的用户体验：

| 平台 | 主要文件 | 支持内容 |
|------|---------|----------|
| 桌面端 | `app.css`, `workspace.css`, `editor.css` | 完整的桌面应用程序界面 |
| 移动端 | `mobile.css` | 移动设备上的响应式适配 |
| Web 发布 | `publish.css` | Obsidian Publish 网站的展示样式 |

#### 3. 变量中心化设计

`theme.css` 作为主题变量中心，集中定义所有设计系统参数：

```css
/* 示例：CSS 变量系统的使用 */
:root {
  /* 颜色系统 */
  --text-normal: #......;
  --text-muted: #......;
  --text-highlight: #......;
  
  /* 间距系统 */
  --spacing-xs: 0.5rem;
  --spacing-m: 1rem;
  --spacing-l: 1.5rem;
  --spacing-xl: 2rem;
  
  /* 圆角系统 */
  --radius-s: 2px;
  --radius-m: 4px;
  --radius-l: 8px;
  
  /* 字体系统 */
  --font-sm: 0.875rem;
  --font-base: 1rem;
  --font-lg: 1.25rem;
}
```

这种设计便于主题定制和深色模式的切换，开发者只需修改变量文件即可实现主题的全局变更。

2. 复制需要的 CSS 文件内容
3. 打开 Obsidian 设置 → 外观 → CSS 代码片段

## 技术亮点

### 架构设计亮点

#### 1. CSS 变量中心化系统

`theme.css` 文件集中管理所有设计变量，构成了完整的设计系统基础：

```css
:root {
  /* 语义化颜色变量 */
  --text-normal: #2d333b;
  --text-muted: #768390;
  --text-faint: #b1bac4;
  
  /* 背景颜色层次 */
  --bg-primary: #ffffff;
  --bg-secondary: #f6f8fa;
  --bg-tertiary: #eaeef2;
  
  /* 功能颜色 */
  --accent-primary: #0969da;
  --accent-secondary: #8250df;
  
  /* 间距系统 */
  --space-inline: 0.375rem;
  --space-inline-xs: 0.125rem;
  --space-inline-sm: 0.25rem;
  --space-inline-m: 0.375rem;
  --space-inline-lg: 0.5rem;
  
  /* 字体规格 */
  --font-small: 0.75rem;
  --font-smaller: 0.6875rem;
}
```

这种设计的核心价值在于：修改主题只需更改变量值，无需遍历整个代码库。

#### 2. 功能模块化拆分

项目将样式代码拆分为 11 个独立模块，每个模块职责明确：

| 模块分类 | 包含文件 | 职责 |
|---------|---------|------|
| 基础框架 | `app.css` | 应用外壳和窗口框架 |
| 布局系统 | `workspace.css`, `sidebar.css` | 页面布局和侧边栏 |
| 内容编辑 | `editor.css`, `formatting.css` | 编辑器和文本格式 |
| 特色功能 | `canvas.css`, `graph.css` | 画布和关系图 |
| 平台适配 | `mobile.css`, `publish.css` | 移动端和发布平台 |
| 视觉基础 | `theme.css` | 主题变量系统 |
| 插件支持 | `core-plugins.css` | 核心插件样式 |

#### 3. 零依赖架构

项目采用零依赖设计，具有以下优势：

- **安装即用**：下载后立即可用，无需任何配置
- **版本稳定**：不受第三方库更新影响
- **安全可靠**：不存在供应链安全风险
- **轻量高效**：无需加载额外的依赖包

#### 4. 响应式设计实现

`mobile.css` 实现了完善的移动端适配：

```css

  }
```
