## 项目简介

UVL (Unified Variability Language，统一变异性建模语言) 是一个专注于软件产品线变异性建模的领域特定语言 (DSL) 项目。该项目由 uvl-ai 团队开发和维护，旨在提供一种人类友好、形式化、供应商无关的变异性建模解决方案。

### 核心概念

- **Features (特征)**: 变异性模型的基本构建块，代表产品线中的可配置选项
- **Feature Groups (特征组)**: 表示互斥特征组，组内特征不能同时选中
- **Constraints (约束)**: 表达特征间的复杂关系，支持 requires、excludes、alternative、or 等约束类型

### 设计理念

| 理念 | 说明 |
|------|------|
| **人类友好** | 清晰直观的语法，降低学习成本 |
| **形式化** | 具备数学定义，支持形式化验证 |
| **供应商无关** | 可移植，跨平台使用 |
| **可扩展** | 支持用户定义约束和规则 |
| **工具无关** | 解耦具体工具实现 |

### 架构设计评估

**架构模式**: 分层模块化架构

```
表现层 (CLI/LSP) ──→ 业务层 (UVL/Serializer/Validator) ──→ 数据层 (AST/Parser)
```

**架构优点**:

- 模块职责清晰，耦合度低
- 支持多种使用方式（库/API/CLI）
- 解析器可替换（ANTLR4/Lark 双解析器）

**架构特点**:

- 面向对象与函数式混合编程范式
- 动态类型 + 静态类型标注（mypy 检查）
- Visitor 模式用于 AST 遍历处理

### 快速使用示例

#### Python API 使用

```python
from uvl import UVL, UVLDocument

# Load and parse a UVL file
document = UVL.parse_file("model.uvl")

# Access features
for feature in document.features:
    print(feature)
    
# Modify the model
document.features.add(Feature("NewFeature"))

# Serialize back to UVL
print(document.to_uvl())
```

#### 命令行使用

