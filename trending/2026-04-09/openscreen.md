## 项目简介

**openscreen** 是一个开源免费的屏幕录制和演示制作工具，旨在为创作者提供专业级的屏幕录制体验。该项目由开发者 @siddharthvaddem 创建，定位为 Screen Studio 的开源替代品，核心特点是完全免费、无水印、可用于商业用途，且无任何订阅费用。

### 核心定位

本项目是一个跨平台桌面应用程序，采用现代化的技术栈构建。通过使用 Tauri 2.0 框架，实现了比传统 Electron 应用更轻量的方案，同时保证了跨平台兼容性（支持 Windows、macOS、Linux）。项目目前处于早期开发阶段（版本 0.0.1），但技术架构设计合理，具有良好的扩展潜力。

### 项目类型

这是一个跨平台桌面应用——屏幕录制和演示制作工具，具体包含以下特性：

- **Screen Studio 替代品**：开源免费的屏幕录制软件
- **Tauri + Svelte 架构**：使用现代跨平台技术栈
- **商业友好**：无水印、可商用、无订阅
- **Electron 替代方案**：使用 Tauri 实现更轻量的桌面应用

## 技术亮点

### 亮点一：Tauri 2.0 现代化架构 ⭐⭐⭐⭐⭐

```toml
# 库类型支持多种构建目标
[lib]
name = "openscreen_lib"
crate-type = ["staticlib", "cdylib", "rlib"]
```

**评价**：Tauri 2.0 是该项目的核心技术选型，相比 Electron 提供了显著的性能优势。项目采用 Rust 作为后端语言，通过 IPC 桥接与 Svelte 前端通信，实现了高效的双向数据传递。系统托盘功能（tray-icon）已配置，为桌面集成提供了良好基础。

### 亮点二：Svelte 5 响应式系统 ⭐⭐⭐⭐

```svelte
<!-- Svelte 5 组件示例 -->
<script>
  let count = $state(0);
  
  function increment() {
    count++;
  }
</script>

<button onclick={increment}>
  Count: {count}
</button>
```

**评价**：使用最新的 Svelte 5 Runes API，采用 `$state` 等新特性实现响应式状态管理。这是当前前端框架领域最现代的方案之一，代码简洁且性能优异。

### 亮点三：模块化 Rust 后端设计 ⭐⭐⭐⭐


**评价**：命令模块化设计便于扩展和维护。通过 Tauri 的命令系统，前端可以方便地调用 Rust 后端功能，这种架构在保证类型安全的同时也提供了良好的扩展性。
