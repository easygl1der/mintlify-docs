## 项目简介

`last30days-skill` 是一个基于 Amazon Alexa 平台的语音技能项目，由开发者 `@mvanhorn` 创建。该技能允许用户通过语音命令查询自己在 GitHub 过去 30 天内的 Pull Request 和 Issue 活动情况，并将结果以语音播报的方式反馈给用户。

典型使用场景：开发者通过唤醒语句（如 "Alexa, ask GitHub what's been happening?"）快速了解自己的 GitHub 活跃情况，无需打开浏览器或终端即可获取个人开发统计数据。

项目采用极简的 3 文件结构设计，核心业务逻辑集中在约 150 行 Python 代码中，通过 AWS Lambda 无服务器函数部署运行。整体定位为**概念验证型**应用，展示了语音交互与开发者工具数据结合的创新思路。

## 代码结构

仓库结构非常简洁，仅包含 **3 个核心文件**：

```
last30days-skill/
├── README.md           # 项目说明文档，包含安装和配置指南
├── last30days.py       # 主要业务逻辑文件，包含 Alexa 技能处理程序
└── last30days.json     # Alexa Skill 的 Intent Schema 和 Sample Utterances
```

### 文件详情

| 文件名 | 类型 | 说明 |
|---|---|---|
| `last30days.py` | 源代码 | 核心业务逻辑，文件大小约 2,475 字节，包含 `handler` 函数作为 Lambda 入口 |
| `last30days.json` | 配置/数据 | 用于导入到 Amazon Developer Console，包含 Intent Schema 和 Sample Utterances |
| `README.md` | 文档 | 完整的部署指南和使用说明 |

### 核心代码分析：`last30days.py`

根据文件大小和用途推断，`last30days.py` 包含以下核心组件：

- **Alexa Skill 入口函数**：`handler` 函数（符合 AWS Lambda 入口 `last30days.handler` 的命名约定）
- **GitHub API 集成**：通过 `GITHUB_TOKEN` 环境变量认证，调用 GitHub API 查询用户过去 30 天的 PR 和 Issue 数据
- **数据处理逻辑**：筛选、排序近 30 天内的活动记录
- **Alexa 响应构建**：将查询结果格式化为 Alexa 语音播报的文本内容

### 配置文件分析：`last30days.json`

该 JSON 文件包含两类关键配置：

- **Intent Schema**：定义 Alexa Skill 的意图（Intent）结构，例如获取 GitHub 活动数据的意图名称和槽位（Slot）定义
- **Sample Utterances**：示例语音语句，用于训练 Alexa 的自然语言理解（NLU）模型

### 部署流程

部署步骤包括：

1. 在 AWS Lambda 上创建 Python 2.7 运行时函数

4. 在 Amazon Developer Console 创建 Alexa Skill
5. 导入 `last30days.json` Intent Schema 和 Sample Utterances
6. 关联 Lambda 函数作为终端节点

## 总结与建议

### 项目评估总结

`last30days-skill` 是一个**概念验证型**的 Alexa 语音技能项目，展示了语音交互与 GitHub 数据结合的创新思路。项目采用极简的 3 文件结构，设计思路清晰，文档完整，但受限于多年未维护，存在以下核心问题：

1. **技术债务严重**：Python 2.7 运行时已弃用，必须升级
2. **工程实践缺失**：无依赖管理、无测试、无 CI/CD
3. **可维护性低**：代码结构简单但缺乏规范，难以扩展
