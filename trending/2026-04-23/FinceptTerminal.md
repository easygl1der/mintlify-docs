### 5.2 依赖安全风险评估

金融应用依赖分析需重点关注以下风险点：

| 风险类型 | 描述 | 风险等级 | 缓解措施 |
|---------|------|---------|---------|
| **过时依赖** | 第三方库版本过旧，存在已知漏洞 | 🟡 中 | 定期执行 `npm audit` / `pip-audit` |
| **间接依赖漏洞** | 传递依赖引入的安全问题 | 🟡 中 | 使用 `npm audit` 分析依赖树 |
| **许可证合规** | 部分依赖存在许可限制 | 🟡 中 | 使用 `fossology` 进行许可证扫描 |
| **供应链攻击** | 依赖包被恶意篡改 | 🔴 高 | 锁定依赖版本，使用私有镜像 |
| **ABI 不兼容** | 底层库版本冲突 | 🟢 低 | 使用虚拟环境或容器隔离 |

### 5.3 依赖管理最佳实践

```bash
# Python 项目
pip freeze > requirements.lock          # 锁定精确版本
pip-compile requirements.in             # 使用 pip-tools 管理依赖

# Node.js 项目
npm ci                                  # 基于 package-lock.json 全新安装
npm audit                               # 安全审计
```
