# MDX 组件手册

Mintlify 在标准 Markdown 基础上扩展了一系列内置 UI 组件，用于提示、卡片、代码切换等场景。所有组件都可以直接在 MDX 文件中使用，无需导入。

`★ Insight ─────────────────────────────────────`**
Mintlify 组件本质上是 **JSX React 组件**——它们可以嵌套、组合、传参。学会组合使用这些组件，可以让文档的"信息密度"大幅提升，同时保持页面的视觉清晰度。
`─────────────────────────────────────────────────`

## Callout — 重点提示框

用于强调重要信息，有 5 种类型：

```mdx
<Note>这是一条普通提示。</Note>

<Tip>这是一个技巧提示。</Tip>

<Warning>这是一个警告。</Warning>

<Danger>这是一个危险操作警告。</Danger>

<Info>这是一个一般信息。</Info>
```

**效果示例：**

<Note>这是一条普通提示。</Note>

<Tip>这是一个技巧提示。</Tip>

<Warning>这是一个警告。</Warning>

### 带标题的 Callout

```mdx
<Note title="注意">带标题的提示框，内容在后面。</Note>
```

<Note title="注意">带标题的提示框，内容在后面。</Note>

### Callout 中使用 Markdown

Callout 内部**支持 Markdown 语法**：

```mdx
<Note title="支持的语法">

- **加粗文本**
- `行内代码`
- [链接](https://example.com)

</Note>
```

<Note title="支持的语法">

- **加粗文本**
- `行内代码`
- [链接](https://example.com)

</Note>

## Card — 单卡片

用于突出展示单个项目或链接：

```mdx
<Card title="Card Title" href="/concepts/overview">
  卡片描述文字，可以写多行内容。
</Card>
```

**效果：**

<Card title="Card Title" href="/concepts/overview">
  卡片描述文字，可以写多行内容。
</Card>

## Cards — 多卡片网格

自动将多个卡片排列成网格布局：

```mdx
<Cards>
  <Card title="卡片1" href="/concepts/overview">
    第一个卡片的内容
  </Card>
  <Card title="卡片2" href="/concepts/markdown-best-practices">
    第二个卡片的内容
  </Card>
  <Card title="卡片3" href="/how-to/deploy-with-mintlify">
    第三个卡片的内容
  </Card>
</Cards>
```

**效果：**

<Cards>
  <Card title="卡片1" href="/concepts/overview">
    第一个卡片的内容
  </Card>
  <Card title="卡片2" href="/concepts/markdown-best-practices">

  </Card>
  <Card title="卡片3" href="/how-to/deploy-with-mintlify">
    第三个卡片的内容
  </Card>
</Cards>

## Steps — 分步骤指南

用于展示顺序流程（安装、部署等）：

```mdx
<Steps>

### 第一步：安装依赖

运行以下命令安装项目依赖。

```bash
npm install
```

### 第二步：配置环境

复制 `.env.example` 到 `.env` 并填写配置。

### 第三步：启动服务

```bash
npm run dev
```

</Steps>
```

**效果：**

运行以下命令安装项目依赖。

```bash

```

### 第二步：配置环境

### 第三步：启动服务

```bash
npm run dev
```

</Steps>

## CodeGroup — 多语言代码切换

同一个代码示例有多个语言/框架版本时，使用 CodeGroup 切换：

````mdx
<CodeGroup>

```bash
npm install mint
```

```python
pip install mint
```

</CodeGroup>
````

**效果：**

<CodeGroup>

```bash
npm install mint
```

```python
pip install mint
```

</CodeGroup>

## Accordion — 可折叠面板

用于存放不常需要但又需要保留的补充信息：

```mdx
<Accordion title="点击展开详细内容">

这里是折叠起来的内容。

- 可以放列表
- 可以放代码块

```js
const example = "代码也没问题";
```

</Accordion>
```

**效果：**

<Accordion title="点击展开详细内容">

这里是折叠起来的内容。

- 可以放列表
- 可以放代码块

```js
const example = "代码也没问题";
```

</Accordion>

## Tabs — 横向标签切换

适用于"这个功能在多个平台都支持"的场景：

```mdx
<Tabs>

<Tab title="macOS">

```bash
brew install mint
```

</Tab>

<Tab title="Windows">

```powershell
scoop install mint
```

</Tab>

<Tab title="Linux">

```bash
curl -L https://mintlify.com/install.sh | bash
```

</Tab>

</Tabs>
```

**效果：**

<Tab title="macOS">

```bash
brew install mint
```

</Tab>

<Tab title="Windows">

```powershell
scoop install mint
```

</Tab>

<Tab title="Linux">

```bash

```

</Tab>

</Tabs>

## 组件组合示例

实际使用中，组件可以嵌套组合：

```mdx
<Steps>

### 准备工作

<Note title="前提条件">

- Node.js 18+
- GitHub 账号
- 已有 Git 仓库

</Note>

### 安装并部署

<Cards>
  <Card title="方式一：Mintlify Cloud" href="https://mintlify.com">
    最简单的部署方式，一键连接 GitHub
  </Card>
  <Card title="方式二：GitHub Pages" href="/how-to/deploy-with-mintlify">
    使用 GitHub Actions 自动部署
  </Card>
</Cards>

### 验证部署

<Warning>部署后需要等待 2-3 分钟才能看到更新。</Warning>

</Steps>
```

## 表格

标准 Markdown 表格完全支持：

```mdx
| 组件 | 用途 | 常用场景 |
|------|------|---------|
| Callout | 强调信息 | 警告、提示、技巧 |
| Card | 突出项目 | 功能介绍、导航 |
| Steps | 顺序流程 | 安装指南、教程 |
| CodeGroup | 代码切换 | 展示多语言示例 |
```

## 进阶：自定义组件

如果内置组件不满足需求，可以在 `snippets/` 目录下创建自定义 React 组件：

1. 在 `snippets/custom-component.mdx` 中定义组件
2. 在需要使用的页面中直接调用（MDX 自动识别 snippets 目录）

这个功能需要 Mintlify Pro 套餐支持。
