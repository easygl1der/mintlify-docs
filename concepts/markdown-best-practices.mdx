# Markdown 最佳实践

遵循这些规范让文档更易读、易维护。

## 标题层级

使用单一 `#` 作为一级标题，避免跳级：

```md
# 一级标题
## 二级标题
### 三级标题
```

## 代码块

始终指定语言：

```md
```javascript
const hello = 'world';
```
```

## 列表

- 使用 `-` 或 `*` 作为无序列表标记
- 保持缩进一致
- 列表项不要过长，必要时拆分成多行

## 链接和图片

```md
[链接文字](https://example.com)
![图片描述](../images/example.png)
```

## Frontmatter

每个 MDX 文件顶部应包含：

```mdx
---
title: "页面标题"
description: "页面描述"
---
```

## 表格

对齐列，使用表头分隔行：

```md
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```
