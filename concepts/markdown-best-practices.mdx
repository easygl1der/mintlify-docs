# Markdown 最佳实践

本指南综合了 [Markdown Guide](https://www.markdownguide.org/basic-syntax/)、[Technical Writing Markdown Best Practices](https://technicalwritingmp.com/docs/markdown-best-practices/)、[Google Markdown Style Guide](https://google.github.io/styleguide/docguide/style.html) 及 [CommonMark 社区讨论](https://talk.commonmark.org/t/markdown-best-practices/3115) 的最佳实践。

> **核心原则**：Markdown 应当能够以纯文本形式直接发布，无需标签标记。可读性第一。

## 标题层级

### 层级结构

使用 ATX 风格（`#`），严禁使用 Setext 风格（`==` / `--`）：

```md
# 文档标题（H1，每个文档只能有一个）
## 主要章节（H2）
### 子章节（H3）
#### 深层子章节（H4）
```

### 规范

- `#` 与文字之间必须有且仅有一个空格
- 标题前后留空行
- **禁止跳级**（如 H1 后直接跳到 H3）
- 标题文字应独特且描述性强，避免 "Overview"、"Details" 等模糊标题
- 标题文本在文档内不应重复

## 段落与换行

- 段落之间留空行
- **不要缩进**（Markdown 不是 HTML，不需要用空格缩进来表示段落）
- 段内换行：行末加两个或以上空格，或使用 `<br>`
- Google 建议 prose 控制在 **80 字符以内**（链接、表格、标题除外）

```md
第一段文字。

空行分隔新的段落。

这一行末有双空格  
所以它和下一行属于同一段落，但显示为换行。
```

## 代码块

### Fenced Code Blocks（推荐）

```md
```javascript
const hello = 'world';
```
```

始终声明语言，以启用语法高亮。

### 行内代码

```md
Use the `print()` function to output text.
```

### 嵌套列表中的代码块

列表项内的代码块需要 **8 个空格** 缩进：

```md
1. Step one
2. Step two
   ```bash
   # 这里需要 8 个空格缩进
   echo "Hello"
   ```
```

### 转义反引号

单反引号内包含反引号：`` `code containing `backticks` around` ``

## 列表

### 无序列表

- 统一使用 `-`、`*` 或 `+` 中的**一种**（不要混用）
- 嵌套缩进 **4 个空格**

```md
- 第一个项目
- 第二个项目
  - 嵌套项目
  - 另一个嵌套项目
- 第三个项目
```

### 有序列表

```md
1. 步骤一
2. 步骤二
   1. 子步骤二-a
   2. 子步骤二-b
3. 步骤三
```

行首句号需转义（防止被识别为列表）：`1968\.`

### 列表内嵌套内容

列表项可包含其他 Markdown 元素：段落、引用、代码块等。代码块需要 8 空格。

## 链接与图片

### 行内链接

```md
[链接文字](https://example.com)
[链接文字](https://example.com "可选标题")
```

### 引用式链接

URL 定义一次，多处引用：

```md
See [Google's search page][google] for more.

[google]: https://www.google.com
```

### 链接文字规范

- 使用描述性锚文本，**禁止** "点击这里"、"here" 等泛泛描述
- 每个链接应能独立说明目的地

```md
Correct:   [Python 官方文档](https://docs.python.org)
Incorrect: [点击这里](https://docs.python.org)
```

### 图片

```md
![Logo](https://example.com/logo.png)
![Logo](https://example.com/logo.png "可选标题")
```

alt 文本应描述图片**内容**，而非外观。

### 图片链接

```md
[![Logo](logo.png)](https://example.com)
```

## 强调与加粗

| 效果 | 语法 | 备注 |
|------|------|------|
| 加粗 | `**text**` 或 `__text__` | 推荐使用 asterisks |
| 斜体 | `*text*` 或 `_text_` | 代码文档中优先用 asterisks |
| 加粗+斜体 | `***text***` | 组合强调 |

```md
**加粗文本**
*斜体文本*
***组合强调***
Mid-word: a***very***important***thing***
```

## 引用块（Blockquotes）

```md
> 这是引用块。
> 可以跨多行。
>
> 引用块内也可以有多个段落（空行前加 `>`）。

> 嵌套引用：
>> 这是嵌套的引用块。
```

### 规范

- 引用块用于引用原文、标注提示或高亮说明
- 内容应简洁，过长的引用会失去重点
- 引用块内可以包含其他 Markdown 元素

## 表格

### 基本语法

```md
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容 | 内容 | 内容 |
```

### 列对齐

```md
| 左对齐   | 居中     | 右对齐 |
|:--------|:--------:|------:|
| Cell    | Cell     |   Cell|
```

- `:---` — 左对齐
- `:---:` — 居中
- `---:` — 右对齐

### 单元格内含管道符

使用 HTML 实体 `&#124;` 或反斜杠转义 `\|`。

### 规范

- 表格仅在真正需要tabular展示时使用
- 长 URL 在表格内使用引用式链接保持可读性
- 对齐管道符便于人工编辑

## 分隔线（Horizontal Rules）

```md
内容

---

更多内容（在分隔线之后）
```

- 使用 `***`、`---` 或 `___` 独立一行
- 分隔线前后留空行
- 数量上三个 asterisk `***` 是常见约定
- 使用节制——它标志着重要的章节分隔

## 转义字符

反斜杠 `\` 用于转义特殊字符，使其按字面显示：

```md
\*not italic\*
\# not a heading
\\  literal backslash
```

可转义的字符：

```
\  `  *  _  { }  [ ]  < >  ( )  #  +  -  .  !  |
```

## 常见陷阱

| # | 陷阱 | 正确做法 |
|---|------|---------|
| 1 | 段落用空格缩进 | 不缩进，留空行分隔 |
| 2 | 标题跳级（H1→H3无H2） | 保持连续层级 |
| 3 | 列表混用 `-`、`*`、`+` | 选定一种，全程统一 |
| 4 | 链接文字用"点击这里" | 使用描述性锚文本 |
| 5 | 代码块不声明语言 | 始终声明语言启用高亮 |
| 6 | 表格管道符不对齐 | 对齐以便人工编辑 |
| 7 | 特殊字符未转义 | 在有语义的上下文中注意转义 |
| 8 | 滥用 HTML | 优先使用标准 Markdown，HTML 仅在必要时使用 |
| 9 | prose 行无长度控制 | 尽量控制在 80 字符以内 |
| 10 | 忘记 Frontmatter | MDX 文件顶部必须包含 YAML frontmatter |

## 文档结构模板

参考 Google Style Guide，标准 Markdown 文档结构：

```md
---
title: "文档标题"
description: "简短描述"
---

# 文档标题

1-3 句话介绍背景。

## 主要章节（H2）

### 子章节（H3）

内容...

## 相关链接

- [Related Doc](link)
- [Another Doc](link)
```
