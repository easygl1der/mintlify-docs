## 项目简介

claude-mem 是一个专注于为大语言模型（LLM）应用提供高效内存管理和上下文处理的 Python 库，其核心创新在于通过智能的上下文压缩和检索机制，显著减少LLM的token消耗同时保持信息完整性。

**核心价值定位：**

- **上下文压缩**: 使用智能算法压缩对话历史，减少token使用量
- **语义检索**: 基于向量嵌入的语义搜索，快速定位相关信息
- **分层存储**: 多级存储策略平衡访问速度和存储成本
- **动态窗口**: 根据任务复杂度动态调整上下文窗口大小

**典型使用场景：**

```python
# 场景1：基础内存管理
from claude_mem import MemoryManager

mem = MemoryManager(max_tokens=4000)
mem.add("用户询问了关于Python编程的问题")
mem.add("我解释了Python的基本特征和用途")
context = mem.get_context()  # 自动管理token数量

# 场景2：带压缩的内存管理
from claude_mem import CompressedMemoryManager

mem = CompressedMemoryManager(
    max_tokens=2000,
    compression_ratio=0.3,  # 保留30%的原始信息
    compression_method="semantic"  # 语义压缩
)

# 场景3：向量检索增强
from claude_mem import VectorEnhancedMemory

mem = VectorEnhancedMemory(
    embedding_model="sentence-transformers/all-MiniLM-L6-v2",
    similarity_threshold=0.7
)
mem.add_document("长文档内容...")
relevant_chunks = mem.search("关于Python的特征描述")
```

## 技术亮点

### 1. 智能上下文压缩

```
# 基础压缩使用
from claude_mem import CompressedMemoryManager

mem = CompressedMemoryManager(
    max_tokens=1000,
    compression_ratio=0.4  # 保留40%的信息
)

# 添加长对话历史
for i in range(50):
    mem.add(f"用户消息 {i}: 详细的问题描述...")
    mem.add(f"AI响应 {i}: 详细的解答内容...")

# 获取压缩后的上下文
context = mem.get_context()
# 原本可能需要8000+ tokens，现在只需要约1000 tokens
```

**优势：** 通过语义理解进行压缩，保留关键信息同时大幅减少token使用量。

### 2. 向量化语义检索

```python
# 向量增强内存管理
from claude_mem import VectorEnhancedMemory
from sentence_transformers import SentenceTransformer

# 初始化向量内存
mem = VectorEnhancedMemory(
    embedding_model="sentence-transformers/all-MiniLM-L6-v2",
    index_type="flat"  # 或 IVF, HNSW 等
)

# 添加文档内容
documents = [
    "Python 是一种解释型、面向对象的高级编程语言",
    "它由吉多·范罗苏姆在1989年创建",
    "Python 设计哲学强调代码的可读性和简洁性"
]
mem.add_documents(documents)

# 语义搜索
results = mem.search("谁创建了Python")
# 返回最相关的文档片段
```

**优势：** 利用向量嵌入进行语义搜索，比传统关键词匹配更准确地找到相关信息。

### 3. 分层存储策略

```python
# 分层内存管理
from claude_mem import HierarchicalMemoryManager

mem = HierarchicalMemoryManager(
    # 短期存储：高速访问，容量有限
    short_term_capacity=1000,  # tokens
    # 长期存储：容量大，访问较慢
    long_term_storage="redis://localhost:6379",
    # 自动迁移策略
    migration_threshold=0.7  # 使用率超过70%时迁移
)

# 使用过程
mem.add("频繁访问的信息")  # 存储在短期存储
mem.add("较少访问的档案信息")  # 可能直接存储在长期存储
```

**优势：** 根据访问频率自动在不同存储层之间迁移数据，优化性能和成本。

### 4. 动态上下文窗口

```python
# 动态窗口大小调整
from claude_mem import AdaptiveMemoryManager

mem = AdaptiveMemoryManager(
    base_window=2000,      # 基础窗口大素
    max_window=8000,       # 最大窗口大小
    complexity_threshold=0.8  # 复杂度阈值
)

# 简单查询使用较小窗口
simple_query = "什么是Python?"
context = mem.get_context_for_query(simple_query)
# 可能只使用基础窗口的一部分

# 复杂查询使用较大窗口
complex_query = "详细解释Python的元类机制及其实际应用场景"
context = mem.get_context_for_query(complex_query)
# 可能自动扩展到更大的窗口以容纳更多上下文
```

**优势：** 根据查询复杂度动态调整上下文窗口，在保证性能的同时提供足够的上下文信息。

`thedotmack/claude-mem` 是一个**创新性强、实用价值高**的 LLM 内存管理库。项目在上下文压缩、向量化检索和分层存储方面表现出色，为解决LLM应用中的核心瓶颈提供了有效方案。

尽管项目目前存在一定的资源消耗和启动延迟，但这些都是其实现先进功能的必要代价。对于需要处理长对话、大量文档或复杂上下文的LLM应用开发者，该项目提供了值得考虑的解决方案，特别是在token成本和上下文限制是主要考虑因素的场景中。
